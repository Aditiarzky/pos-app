Tujuan tiap file:

- `uml-use-case.puml`
  - dasar Use Case Diagram utama
- `uml-activity-sales.puml`
  - dasar Activity Diagram untuk alur penjualan
- `uml-activity-purchase.puml`
  - dasar Activity Diagram untuk alur pembelian / kulakan
- `uml-activity-return.puml`
  - dasar Activity Diagram untuk alur retur customer
- `uml-activity-product.puml`
  - dasar Activity Diagram untuk tambah, ubah, dan hapus produk
- `uml-activity-trash-cleanup.puml`
  - dasar Activity Diagram untuk pembersihan trash
- `uml-activity-trash-restore.puml`
  - dasar Activity Diagram untuk restore data dari trash
- `uml-sequence-sales.puml`
  - dasar Sequence Diagram untuk transaksi penjualan tunai dan QRIS
- `uml-sequence-purchase.puml`
  - dasar Sequence Diagram untuk buat, ubah, dan batalkan pembelian
- `uml-sequence-return.puml`
  - dasar Sequence Diagram untuk retur refund, credit note, exchange, dan pembatalan retur
- `uml-sequence-trash.puml`
  - dasar Sequence Diagram untuk lihat trash, restore, force delete, dan cleanup
- `uml-sequence-notifications.puml`
  - dasar Sequence Diagram untuk pengambilan dan pembacaan notifikasi
- `uml-class-overview.puml`
  - dasar Class Diagram inti sistem
- `uml-presentation-guide.md`
  - panduan memilih dan menyederhanakan diagram untuk presentasi skripsi

Catatan:

- Draft ini dibuat dari implementasi sistem yang ada saat ini.
- Jika ingin diagram yang lebih lengkap, tahap berikutnya bisa diarahkan ke:
  - pemecahan use case besar menjadi diagram per modul
  - pembuatan versi class diagram yang lebih ringkas untuk slide
  - pembuatan narasi bab perancangan berdasarkan tiap diagram
